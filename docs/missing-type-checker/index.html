The missing runtime type checker for Python

Python's typing module is a great tool for avoiding dumb type errors. But it doesn't make python a typed language. The interpreter is still the same duck-typed goo that we all love.

Background

The interpreter doesn't have any special knowledge about the types in the typing module. 

names: typing.List[str] = ["guido", "tim", "barry"]

From the perspective of the interpreter, name is a list.
